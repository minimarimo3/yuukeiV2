# Daihon言語リファレンス

この文書は、World Pack作者がDaihonの台本を書くためのリファレンスです。プログラミング経験がなくても読めるように、実際に使う形を中心に説明します。

## 1. Daihonとは

Daihonは、YuukeiのWorld Packに入れる「台本」です。
どの合図で、どの住人が、どんな条件のときに、どんなセリフや動きをするかを書きます。
World PackとDaihonの役割分担は、[05-world-pack-and-daihon.md](05-world-pack-and-daihon.md)も参照してください。
この文書では、Daihonファイルに何を書くかだけを扱います。

## 2. 最小の台本

Daihonファイルは、イベント、シーン、話者、セリフの順に書きます。

```daihon
## 起動

### 朝のあいさつ
話者: ゆうけい
「おはようございます。」
```

`## 起動` はイベント名です。アプリやWorld Pack側から「起動」というイベントが実行されると、この中のシーンが候補になります。

`### 朝のあいさつ` はシーン名です。1つのイベントの中に、複数のシーンを書けます。

`話者: ゆうけい` は、このシーンで主に話す住人の名前です。

`「おはようございます。」` はセリフです。セリフは必ず `「」` で囲みます。

## 3. ファイル構造

Daihonファイルは、上から順にイベントを書き、その中に前提条件、初期値、シーンを書きます。

```daihon
## 起動

前提条件:
※（入力#停止 = はい）→おわり

初期値:
全体#起動回数 = 0

### 通常の起動
話者: ゆうけい
「今日もよろしくお願いします。」
```

`前提条件:` は、イベント全体を実行してよいかを先に判定する場所です。ここでは `※（条件）→おわり` の形だけを書けます。条件が「はい」になると、イベント全体がそこで終わります。

`初期値:` は、イベントを使う前に変数へ初期値を入れる場所です。変数については「7. 変数」で説明します。

`###` から始まる行がシーンです。シーンには、メタデータ、セリフ、条件分岐、ジャンプなどを書けます。

## 4. シーンのメタデータ

シーンの先頭には、選ばれ方や話者を決めるメタデータを書けます。

```daihon
## デスクトップ反応

### 朝のクリック反応
合図: ＠クリック または ＠ダブルクリック
条件: 時 6~11
優先度: 10
重み: 3
クールダウン: 1日
話者: ゆうけい
「朝から元気ですね。」
```

`合図:` は、このシーンが反応する合図です。合図名の前には `＠` を付けます。複数の合図は `または` でつなげます。`かつ` は使えません。

```daihon
合図: ＠クリック または ＠タップ
```

`条件:` は、シーンを選ぶ前に確認する条件です。`※` は付けず、条件の中身だけを書きます。

```daihon
条件: 入力#時間帯 = 「朝」
```

`優先度:` は、同じ合図に反応できるシーンの中で、どれを先に選ぶかを決める数値です。数値が大きいほど優先されます。

`重み:` は、同じ優先度のシーンが複数あるときの抽選されやすさです。1以上の数値を書きます。

`クールダウン:` は、一度選ばれたあと、しばらく同じシーンを選ばないための時間です。`秒`、`分`、`時間`、`日`、または `s`、`m`、`h`、`d` を使えます。

```daihon
クールダウン: 30秒
クールダウン: 5分
クールダウン: 2時間
クールダウン: 1日
クールダウン: 10s
クールダウン: 3m
クールダウン: 1h
クールダウン: 1d
```

`話者:` は、シーンの主な話者です。セリフごとに話者を書かない場合、この話者が使われます。

シーンは、次の順で選ばれます。

1. 合図が一致するシーンを探します。
2. 条件が「はい」になるシーンだけを残します。
3. クールダウン中のシーンを除きます。
4. 優先度が一番高いシーンを残します。
5. 同じ優先度の中から、重みによって抽選します。

合図つきのシーンが1つも選べなかったとき、`合図:` も `条件:` もないシーンは受け皿として使われます。合図なしでイベントを実行したときは、合図のないシーンが通常の候補になります。

```daihon
## デスクトップ反応

### クリック反応
合図: ＠クリック
話者: ゆうけい
「クリックしましたね。」

### 受け皿
話者: ゆうけい
「今は特別な反応がありません。」
```

## 5. セリフと埋め込み

セリフは `「」` で囲みます。

```daihon
## 起動

### あいさつ
話者: ゆうけい
「こんにちは。」
```

セリフの中には、変数を埋め込めます。埋め込みには全角の `＜＞` を使います。

```daihon
## 起動

### 入力つき
話者: ゆうけい
「今は＜入力#時間帯＞です。」
```

普通の変数も埋め込めます。

```daihon
## 起動

初期値:
ユーザー発言 = 「おはよう」

### 復唱
話者: ゆうけい
「さっき、＜ユーザー発言＞と言いましたね。」
```

セリフの中で記号そのものを書きたいときは、同じ記号を2つ重ねます。

```daihon
## 表記

### 記号の説明
話者: ゆうけい
「セリフの中に「「かぎかっこ」」や＜＜山かっこ＞＞を書けます。」
```

セリフは複数行にできます。改行もセリフの一部になります。

```daihon
## 手紙

### 長いセリフ
話者: ゆうけい
「おかえりなさい。
今日は少し静かな日ですね。」
```

## 6. 関数呼び出し

セリフの中の `＜＞` は、変数だけでなく関数呼び出しにも使えます。関数は、セリフ以外の動きや外部機能を呼ぶためのものです。

```daihon
## 表情

### 笑う
話者: ゆうけい
＜表情 笑顔＞
「ふふ、嬉しいです。」
```

```daihon
## 動作

### 手を振る
話者: ゆうけい
＜動作 手を振る＞
「いってらっしゃい。」
```

`表情` や `動作` のあとに並ぶ値を、引数と呼びます。引数は、関数に渡す追加情報です。上の例では `笑顔` や `手を振る` が位置引数です。位置引数は、前から順番に意味が決まります。

関数によっては、名前付き引数を受け取るものもあります。名前付き引数は `名前=値` の形です。次は、World PackやYuukei側が `表示` という関数を登録している場合の例です。

```daihon
## 関数の例

### 名前付き引数
話者: ゆうけい
＜表示 テキスト=「おかえりなさい」 秒数=3＞
「画面にも出しておきますね。」
```

使える関数は、World PackやYuukei側が増やせます。Daihon本体は、関数名と引数の形を見て、登録されている関数を呼び出します。

## 7. 変数

変数は、あとで使う値に名前を付けるものです。スコープは、変数の置き場所の種類です。

| 書き方 | 意味 | 例 |
| --- | --- | --- |
| `名前` | イベント内の変数 | `好感度` |
| `_一時` | 一時変数 | `_候補` |
| `全体#名前` | 全体で共有する変数 | `全体#起動回数` |
| `入力#名前` | イベント実行時に外から渡される値 | `入力#時間帯` |
| `住人#actor#名前` | 住人ごとの変数 | `住人#yuu#眠気` |
| `関係#a#b#名前` | 2人の関係にある変数 | `関係#yuu#user#親しさ` |

```daihon
## 変数

初期値:
好感度 = 5
全体#起動回数 = 0

### 表示
話者: ゆうけい
「好感度は＜好感度＞です。」
```

Daihonの値には、数値、文字列、真偽値があります。真偽値は「はい」か「いいえ」の値です。

```daihon
## 型

初期値:
好感度 = 5
名前 = 「ゆうけい」
起きている = はい

### 型の例
話者: ゆうけい
「＜名前＞、起きています。」
```

一度入れた変数の型は固定されます。数値として使い始めた変数には、あとから文字列を入れられません。

組み込み時刻変数は、現在時刻から自動で読める変数です。

| 変数 | 内容 |
| --- | --- |
| `年` | 年 |
| `月` | 月 |
| `日` | 日 |
| `曜日` | 曜日 |
| `週` | 週番号 |
| `時` | 時 |
| `分` | 分 |
| `秒` | 秒 |
| `ミリ秒` | ミリ秒 |

```daihon
## 時刻

### 朝
条件: 時 6~11
話者: ゆうけい
「今は朝ですね。」
```

組み込み時刻変数には代入できません。たとえば `時 = 5` と書くと `E-DHN-SEM-049` になります。別の変数名を使ってください。

## 8. 条件と式

条件分岐は、`※（条件）なら:` で始め、`おわり` で閉じます。

```daihon
## 条件

初期値:
好感度 = 5

### 分岐
話者: ゆうけい
※（好感度 >= 5）なら:
「少し仲良くなりましたね。」
※それ以外:
「これからですね。」
おわり
```

`※あるいは` は、前の条件が「いいえ」だったときに、別の条件を試します。

```daihon
## 条件

初期値:
好感度 = 5

### 複数分岐
話者: ゆうけい
※（好感度 >= 10）なら:
「とても仲良しです。」
※あるいは（好感度 >= 5）:
「いい感じです。」
※それ以外:
「まだこれからです。」
おわり
```

1行だけの条件分岐なら、次のようにも書けます。

```daihon
## 一行条件

初期値:
好感度 = 5

### 一行
話者: ゆうけい
※（好感度 >= 5）→なかよし

### なかよし
話者: ゆうけい
「仲良しです。」
```

比較には、`=`、`!=`、`<`、`<=`、`>=` が使えます。

```daihon
## 比較

初期値:
好感度 = 5

### 比較
話者: ゆうけい
※（好感度 = 5）なら:
「ちょうど5です。」
おわり
```

日本語の後置比較も使えます。

```daihon
## 後置比較

初期値:
好感度 = 5

### 後置
話者: ゆうけい
※（好感度 10未満）なら:
「10未満です。」
おわり
```

使える後置比較は、`未満`、`以下`、`以上`、`超える` です。

`でない` を後ろに付けると、条件を反対にできます。

```daihon
## 否定

初期値:
好感度 = 5

### まだ
話者: ゆうけい
※（好感度 10以上 でない）なら:
「まだ10以上ではありません。」
おわり
```

文字列には、`を含む`、`で始まる`、`で終わる` が使えます。右辺は `「」` で囲む書き方をおすすめします。変数を右辺に使う場合は、左辺、右辺、キーワードの間に空白を入れてください。

```daihon
## 文字列判定

### ファイル名
話者: ゆうけい
※（入力#ファイル名 「レポート」を含む）なら:
「レポートのファイルですね。」
※あるいは（入力#ファイル名 「.png」で終わる）:
「画像ファイルですね。」
※あるいは（入力#ファイル名 「月次」で始まる）:
「月次のファイルですね。」
※それ以外:
「別のファイルですね。」
おわり
```

条件は `かつ`、`または` でつなげられます。

```daihon
## 論理

初期値:
a = はい
b = いいえ

### 論理
話者: ゆうけい
※（（a かつ b）でない）なら:
「両方はい、ではありません。」
おわり
```

算術には、`+`、`-`、`*`、`/`、`%` が使えます。全角の `＋`、`－`、`＊`、`／`、`％` も使えます。

```daihon
## 算術

初期値:
好感度 = 5

### 加算
話者: ゆうけい
好感度 = 好感度＋1
※（好感度 ＊ 2 >= 10）なら:
「十分大きいです。」
おわり
```

整数同士の割り算は、小数点以下を切り捨てます。たとえば `5 / 2` は `2` になります。

時刻の範囲は `9:00~12:00` のように書けます。これは、現在時刻がその範囲に入っているかを調べます。日をまたぐ範囲も使えます。

```daihon
## 時刻範囲

### 夜
条件: 22:00~5:00
話者: ゆうけい
「夜更かし中ですね。」
```

数値の範囲は、左に変数、右に `10~20` のような範囲を書きます。

```daihon
## 数値範囲

初期値:
好感度 = 15

### 範囲
話者: ゆうけい
※（好感度 10~20）なら:
「10から20の間です。」
おわり
```

## 9. 進行制御

`→シーン名` は、同じイベントの別シーンへ移動します。

```daihon
## 進行

### はじめ
話者: ゆうけい
「次へ行きます。」
→つづき

### つづき
話者: ゆうけい
「続きです。」
```

`→おわり` は、イベント全体を終わらせます。

```daihon
## 終了

### すぐ終わる
話者: ゆうけい
「ここで終わります。」
→おわり
```

`→シーンおわり` は、今のシーンだけを終わらせます。

```daihon
## シーン終了

### 途中まで
話者: ゆうけい
「ここまで言います。」
→シーンおわり
「このセリフは実行されません。」
```

## 10. AI連携

AI連携の考え方は、[05-world-pack-and-daihon.md](05-world-pack-and-daihon.md)のDaihonの説明も参照してください。ここでは台本での書き方だけを説明します。

`解釈` は、入力文を選択肢のどれに近いかへ分類する関数です。引数は3つで、順序は「解釈対象、質問、選択肢」です。変数の値を渡すときは、`(入力#発言)` のように `()` で包みます。

```daihon
## 解釈の例

### ユーザー発言
話者: ゆうけい
_意図 = ＜解釈 (入力#発言) 「発言の意図は何ですか」 「挨拶/質問/終了」＞

※（_意図 = 「挨拶」）なら:
「こんにちは。」
※あるいは（_意図 = 「質問」）:
「質問ですね。」
※あるいは（_意図 = 「終了」）:
「またあとで。」
※それ以外:
「うまく読み取れませんでした。」
おわり
```

選択肢は `/`、`／`、`|`、`、`、`,` で区切れます。結果は、選択肢のどれか、または `不明` です。`不明` になったときの枝を必ず用意してください。上の例では `※それ以外` が受け皿です。

`生成` は、短い文をAIに作らせる関数です。生成されたセリフは、その場で現在の話者のセリフとして出ます。第2引数は任意で、生成できなかったときに代わりに言うセリフです。

```daihon
## 生成の例

### 一言
話者: ゆうけい
「おはようございます。」
＜生成 「朝の気分をひとことつぶやく」 「今日もいい天気ですね。」＞
```

## 11. コメント

`$$` または全角の `＄＄` から行末まではコメントです。コメントは台本の実行に影響しません。

```daihon
## コメント

$$ これはコメントです
＄＄ これもコメントです

### 例
話者: ゆうけい
「コメントは表示されません。」
```

## 12. 入力のしかた早見表

よく使う記号は、IMEで次のように変換できます。

| 記号 | 読み方の例 | 用途 |
| --- | --- | --- |
| `※` | こめ | 条件分岐 |
| `→` | やじるし | ジャンプ |
| `＜` `＞` | やまかっこ、しょうなり、だいなり | 埋め込み、関数呼び出し |
| `「` `」` | かぎかっこ | セリフ、文字列 |
| `＠` | あっとまーく | 合図 |
| `＃` または `#` | しゃーぷ | 変数スコープ |
| `〜` または `~` | から、ちるだ | 範囲 |
| `＄＄` または `$$` | どる | コメント |

Daihonでは、日本語入力で書きやすいように、いくつかの全角記号を半角と同じ意味で扱います。ただし、比較の `＜` `＞` は関数呼び出しの記号と重なるため、条件では半角の `<` `>` を使ってください。

## 13. エラーコード一覧

エラーコードは、先頭の文字で種類が分かれます。`E-DHN-*` は実行できないエラー、`W-DHN-*` は動くけれど注意が必要な警告です。

| コード | 意味 | よくある原因と直し方 |
| --- | --- | --- |
| `E-DHN-LEX-001` | セリフまたは文字列が閉じていません。 | `「` に対応する `」` を追加してください。 |
| `E-DHN-LEX-002` | 関数呼び出しまたは埋め込みが閉じていません。 | `＜` に対応する `＞` を追加してください。 |
| `E-DHN-LEX-010` | 使えない文字があります。 | 条件や文の外に余分な記号がないか確認してください。 |
| `E-DHN-LEX-011` | 関数呼び出しの中に使えない文字があります。 | `＜...＞` の中の記号や閉じ忘れを確認してください。 |
| `E-DHN-SYN-001` | イベント見出しがありません。 | ファイルの先頭に `## イベント名` を書いてください。 |
| `E-DHN-SYN-010` | 前提条件に書けない文があります。 | `前提条件:` には `※（条件）→おわり` だけを書いてください。 |
| `E-DHN-SYN-011` | 初期値に書けない文があります。 | `初期値:` には `変数 = 値` だけを書いてください。 |
| `E-DHN-SYN-020` | シーン見出しが必要です。 | セリフや文の前に `### シーン名` を書いてください。 |
| `E-DHN-SYN-030` | `おわり` の対応がありません。 | 条件分岐の始まりと終わりを確認してください。 |
| `E-DHN-SYN-031` | 文として読めません。 | セリフは `「」` で囲み、地の文をそのまま書かないでください。メタデータ行はシーン見出しの直後にまとめてください。 |
| `E-DHN-SYN-040` | `※あるいは` または `※それ以外` の位置が不正です。 | 直前に `※（条件）なら:` の分岐を書いてください。 |
| `E-DHN-SYN-041` | 条件式を読めません。 | `※（...）なら:` の中を確認してください。 |
| `E-DHN-SYN-042` | 条件分岐が閉じていません。 | 分岐の最後に `おわり` を追加してください。 |
| `E-DHN-SYN-043` | `※あるいは` の条件を読めません。 | `※あるいは（条件）:` の形にしてください。 |
| `E-DHN-SYN-044` | `※あるいは` に `:` がありません。 | 行末に `:` を追加してください。 |
| `E-DHN-SYN-045` | 条件分岐の終わりを読めません。 | `※それ以外:` や `おわり` の書き方を確認してください。 |
| `E-DHN-SYN-046` | 1行条件の実行内容を読めません。 | `※（条件）→シーン名` の右側を確認してください。 |
| `E-DHN-SYN-050` | 関数名が空です。 | `＜＞` の中に関数名か変数名を書いてください。 |
| `E-DHN-SYN-060` | 条件式の後ろに余分な文字があります。 | 条件の末尾に不要な語がないか確認してください。 |

## 14. 制限事項

Daihonには、現在いくつかの制限があります。

- リスト型はありません。複数の値を1つの変数にまとめることはできません。
- 地の文はありません。セリフは必ず `「」` で囲んでください。
- イベントを跨ぐジャンプはできません。`→シーン名` は同じイベント内だけです。
- 待ち時間や、シーン内でのランダム分岐はありません。
- クールダウンはアプリを再起動するとリセットされます。
- 比較は同じ型同士だけが有効です。型が違う値を比較すると、警告が出て「いいえ」として扱われます。
