# Daihonをはじめから書く

Daihonは、Yuukeiの住人に「何が起きたとき」「どんな条件なら」「誰が」「何を言い、どう動くか」を教える台本言語です。日本語の見出し、かぎかっこ、条件文を中心に書けます。

この章は、Daihonを初めて書く人向けのチュートリアルです。すべての構文とエラーコードを調べたいときは [Daihon言語リファレンス](../../08-daihon-language-reference.md) を使ってください。

## 1. Daihonの考え方

Daihonは、上から順に必ず全部を再生する小説ではありません。1つの出来事が届くたびに、条件に合うsceneを探して実行する台本です。

```text
RuntimeEventが届く
        ↓
World Packのsignals.allowを確認
        ↓
同じ合図を持つsceneを集める
        ↓
条件・頻度を満たす候補だけ残す
        ↓
最も具体的な候補から1つ選ぶ
        ↓
セリフ・動作・変数更新を実行
```

たとえば、ユーザーが住人をつつくと`avatar.gesture.poke`というcanonical eventが届きます。Daihonでは、その日本語別名`住人_つつく`を使えます。

```daihon
## 住人_つつく

### びっくりする
合図: ＠住人_つつく
話者: ア
「わっ、びっくりしました。」
```

`##`はイベントのまとまり、`###`はscene、`合図:`は反応する出来事、`話者:`は住人、`「」`はセリフです。

## 2. 最小構文を1行ずつ読む

```daihon
## アプリ_起動

### 朝の挨拶
合図: ＠アプリ_起動
条件: 6:00~11:59
頻度: 1日に1回
話者: ア
＜表情 笑顔＞
「おはようございます。」
```

### `## アプリ_起動`

イベントの見出しです。関連するsceneをまとめます。イベント名が届いた合図と同じなら、その中の合図なしsceneも候補になれます。ただし、初心者のうちは各sceneに`合図:`を明示すると、別イベントへsceneを移したときも意図が分かりやすくなります。

### `### 朝の挨拶`

sceneの名前です。同じイベント内で重複できません。診断や実行履歴にも出るため、「挨拶1」より「初回起動の自己紹介」「深夜につつかれた」のように意味が分かる名前を付けます。

### `合図: ＠アプリ_起動`

sceneが反応するsignalです。先頭の`＠`を忘れないでください。複数の合図は`または`でつなげます。

```daihon
合図: ＠住人_つつく または ＠住人_なでる
```

合図に`かつ`は使えません。1つのRuntimeEventが同時に2つのtypeになることはないためです。

### `条件: 6:00~11:59`

sceneを候補に残す条件です。`条件:`には`※`を付けません。条件が偽なら、このsceneは選ばれません。

### `頻度: 1日に1回`

sceneの回数制限です。実行履歴は再起動後も残ります。`一度きり`、`30秒に1回`、`5分に1回`、`2時間に1回`、`1日に1回`などを使えます。

### `話者: ア`

このsceneの既定話者です。World Packのactor ID、または`actors[].speakerAliases`へ登録した名前を使います。`displayName`だけでは話者として解決されません。

### `＜表情 笑顔＞`

関数呼び出しです。全角の`＜＞`で囲みます。この例は現在の話者へ`avatar.expression` commandを出します。

### `「おはようございます。」`

セリフです。地の文はなく、表示する文章は必ず`「」`で囲みます。

## 3. よく使う記号

| 記号 | 役割 | 例 |
| --- | --- | --- |
| `##` | イベント見出し | `## 会話_入力` |
| `###` | scene見出し | `### 挨拶` |
| `＠` | 合図 | `合図: ＠会話_入力` |
| `「」` | セリフ・文字列 | `「こんにちは」` |
| `＜＞` | 関数呼び出し・埋め込み | `＜表情 笑顔＞`、`＜名前＞` |
| `※` | 条件分岐 | `※（条件）なら:` |
| `→` | scene移動・終了 | `→つづき`、`→おわり` |
| `#` | 変数スコープの区切り | `全体#起動回数` |
| `$$` | コメント | `$$ 作者用メモ` |

日本語入力では、`※`は「こめ」、`→`は「やじるし」、`＜＞`は「やまかっこ」、`「」`は「かぎかっこ」で変換できます。

## 4. セリフを書く

### 普通のセリフ

```daihon
「こんにちは。」
```

### 複数行

```daihon
「おかえりなさい。
今日は静かな日ですね。」
```

改行もセリフの一部になります。吹き出しが大きくなりすぎないよう、デスクトップでは短めの文を複数に分ける方が読みやすくなります。

### 変数を埋め込む

```daihon
「＜全体#呼び名＞さん、おかえりなさい。」
```

未定義のスコープ付き埋め込みは空文字になり、警告が出ます。表示前に条件で空文字ではないことを確認すると安全です。

```daihon
※（全体#呼び名 != 「」）なら:
「＜全体#呼び名＞さん、おかえりなさい。」
※それ以外:
「おかえりなさい。」
おわり
```

### Daihonの記号をセリフ内へ書く

`「」`や`＜＞`そのものをDaihonのセリフ内へ出したい場合、同じ記号を2つ重ねます。

```daihon
「「「秘密」」と書いてある箱です。」
「＜＜これは変数ではありません＞＞」
```

### 行だけ話者を変える

```daihon
話者: ア
「おかえりなさい。」
ボ: 「待っていましたよ。」
ア: ＜表情 笑顔＞「二人で留守番していました。」
```

`ボ:`のような短縮名はWorld Packの`speakerAliases`に必要です。話者付きの書き方は表示行にだけ使えます。

## 5. 変数を使う

変数は、数値、文字列、真偽値のどれかを持つ名前付きの箱です。

```daihon
初期値:
全体#起動回数 = 0
全体#呼び名 = 「」
全体#初対面済み = いいえ
```

真偽値は`はい`と`いいえ`です。JSONでは`true`と`false`ですが、Daihonでは日本語で書きます。

### スコープ

| 書き方 | 寿命・用途 | 保存 |
| --- | --- | --- |
| `名前` | イベント内で使う値 | されない |
| `_一時` | scene内の計算結果やAI結果 | されない |
| `入力#名前` | RuntimeEventやResident Homeから届く読み取り専用値 | されない、代入不可 |
| `全体#名前` | Pack全体の進行やユーザー情報 | 再起動後も保存 |
| `住人#actor#名前` | actorごとの状態 | 再起動後も保存 |
| `関係#a#b#名前` | 2者間の関係値 | 再起動後も保存 |

例:

```daihon
全体#起動回数 = 全体#起動回数 + 1
住人#alice#眠気 = 住人#alice#眠気 + 1
関係#alice#user#親しさ = 関係#alice#user#親しさ + 1
_返事 = ＜選択 「はい」 「いいえ」＞
```

一度決まった変数の型は固定です。数値の変数へ文字列を入れると実行エラーになります。

```daihon
全体#起動回数 = 1
$$ 次は型が違うため不可
全体#起動回数 = 「一回」
```

### 組み込み時刻変数

`年`、`月`、`日`、`曜日`、`週`、`時`、`分`、`秒`、`ミリ秒`を参照できます。これらには代入できません。

```daihon
条件:（時 >= 6 かつ 時 <= 11）
```

時刻範囲の方が簡潔です。

```daihon
条件: 6:00~11:59
条件: 22:00~5:00
```

後者のように日付をまたぐ範囲も使えます。

## 6. 条件を書く

条件には、sceneを選ぶ前の`条件:`と、scene実行中の`※`分岐があります。

### sceneの条件

```daihon
### Downloadsの画像
合図: ＠ダウンロード_完了
条件:（入力#ファイル種類 = 「image」 かつ 入力#在席 = 「present」）
話者: ア
「新しい画像が届きましたね。」
```

### scene内の分岐

```daihon
※（入力#ファイル種類 = 「image」）なら:
「画像ですね。」
※あるいは（入力#ファイル種類 = 「archive」）:
「箱のような圧縮ファイルですね。」
※それ以外:
「何か届きました。」
おわり
```

`※あるいは`は必要なだけ追加できます。`※それ以外`は最後に1つだけです。分岐全体を`おわり`で閉じます。

### 比較

| 種類 | 演算子・書き方 | 例 |
| --- | --- | --- |
| 等しい | `=` | `入力#場所 = 「downloads」` |
| 等しくない | `!=` | `全体#呼び名 != 「」` |
| 大小 | `<` `<=` `>=` `>` | `入力#移動距離 >= 100` |
| 日本語の大小 | `未満` `以下` `以上` `超える` | `好感度 10以上` |
| 文字を含む | `を含む` | `入力#ファイル名 「report」を含む` |
| 先頭一致 | `で始まる` | `入力#ファイル名 「月次」で始まる` |
| 末尾一致 | `で終わる` | `入力#ファイル名 「.png」で終わる` |
| 数値範囲 | `開始~終了` | `好感度 10~20` |
| 否定 | `でない` | `（好感度 10以上）でない` |
| 論理積 | `かつ` | `a かつ b` |
| 論理和 | `または` | `a または b` |

異なる型同士の比較は警告になり、偽として扱われます。`入力#移動距離`は数値、`「100」`は文字列なので、次は一致しません。

```daihon
$$ 正しい
条件: 入力#移動距離 >= 100

$$ 誤り: 数値と文字列を比較している
条件: 入力#移動距離 = 「100」
```

### 計算

```daihon
全体#回数 = 全体#回数 + 1
残り = 10 - 全体#回数
倍 = 残り * 2
商 = 5 / 2
余り = 5 % 2
```

整数同士の`5 / 2`は`2`になります。0除算はエラーです。`+`は数値加算のほか、文字列同士の結合にも使えます。

```daihon
_説明 = 「前面アプリは 」 + 入力#アプリ + 「 です」
```

## 7. sceneが複数あるときの選ばれ方

同じ合図に反応するsceneを複数書けます。Daihonは次の順で1つを選びます。

1. 合図が一致するsceneを集める。
2. `条件:`が真のsceneだけ残す。
3. `頻度:`の制限中を除く。
4. 条件が最も具体的なsceneだけ残す。
5. 同じイベントで直前に選ばれたsceneを、他候補があれば除く。
6. 残りから一様に抽選する。

具体性は、トップレベルの`かつ`でつないだ条件数です。

```daihon
### 普通のつつき
合図: ＠住人_つつく
話者: ア
「どうしました？」

### 頭をつつかれた
合図: ＠住人_つつく
条件:（入力#hitZoneId = 「head」）
話者: ア
「髪が乱れてしまいます。」

### 深夜に頭をつつかれた
合図: ＠住人_つつく
条件:（入力#hitZoneId = 「head」 かつ 23:00~4:59）
話者: ア
「こんな時間に、まだ起きていたんですか。」
```

深夜に頭をつつくと、2条件を持つ最後のsceneが最も具体的です。昼に頭をつつくと2番目、それ以外の部位では条件なしの最初が候補になります。

`優先度:`、`重み:`、`クールダウン:`は廃止されています。警告が出て無視されます。

- 特別な状況を優先したい: `条件:`を具体的にする。
- 同じ状況の台詞を増やしたい: 同じ条件のsceneを追加する。
- 回数を抑えたい: `頻度:`を使う。

## 8. 標準の合図と入力

### 標準signal

| Daihon別名 | canonical ID | いつ届くか |
| --- | --- | --- |
| `会話_入力` | `conversation.text` | ユーザーが文章を送った |
| `画面_接続` | `surface.attach` | Surfaceが接続した |
| `アプリ_起動` | `app.startup` | Resident HomeがPackで起動した |
| `生活_定期` | `presence.life_tick` | 生活時計が進んだ |
| `雑談_定期` | `presence.talk_impulse` | 自発的なひとりごとの候補時刻になった |
| `時間帯_変化` | `presence.time_period` | 時間帯が変わった |
| `不在_開始` | `presence.idle.start` | ユーザーの無操作が閾値を超えた |
| `復帰` | `presence.idle.end` | 無操作状態から戻った |
| `端末_スリープ前` | `device.sleep.before` | 端末がスリープへ入る前 |
| `端末_復帰` | `device.wake` | 端末が復帰した |
| `住人_つつく` | `avatar.gesture.poke` | 住人をつついた |
| `住人_なでる` | `avatar.gesture.pat` | 住人をなでた |
| `住人_つまむ` | `avatar.gesture.grab` | 長押しで持ち上げた |
| `住人_おろす` | `avatar.gesture.drop` | 移動後に置いた |
| `住人_歩き終わり` | `stage.walk.ended` | `歩く`が到着・中断した |
| `窓_出現` | `desktop.window.appeared` | 新しいウィンドウが現れた |
| `窓_消滅` | `desktop.window.closed` | ウィンドウが消えた |
| `窓_注目` | `desktop.window.focused` | 前面ウィンドウが変わった |
| `フォルダ_開いた` | `desktop.folder.opened` | 既知カテゴリのフォルダが表示された |
| `ダウンロード_完了` | `desktop.download.completed` | Downloadsへファイルが追加された |

使うsignalは`pack.json`の`signals.allow`にも必要です。

### 共通入力

すべてのイベントでは、少なくとも次を参照できます。

| 変数 | 内容 |
| --- | --- |
| `入力#合図` | canonical event type |
| `入力#イベント種別` | canonical event type |
| `入力#イベントID` | イベントの一意ID |
| `入力#source` | `user`、`device`など発生源 |
| `入力#actorId` | 対象住人がある場合のactor ID |
| `入力#場所` | 対象住人の意味上の現在地 |
| `入力#在席` | `present`または`away` |
| `入力#AI接続` | `dialogue.generate` providerがあれば`はい` |

event payloadの単純な文字列・数値・真偽値は、元のkeyでも`payload.key`でも入力になります。Yuukeiは作者向けの日本語名も追加します。

### よく使うイベント固有入力

| 合図 | 入力 | 内容 |
| --- | --- | --- |
| `会話_入力` | `入力#ユーザー発言` | 入力された文章 |
| `時間帯_変化` | `入力#時間帯` | 時間帯の文字列 |
| `復帰` | `入力#不在分` | 無操作だった分数 |
| `住人_つつく` / `住人_なでる` | `入力#hitZoneId` | `head`、Pack固有IDなど |
| 同上 | `入力#hitZoneLabel` | 部位の表示名 |
| 同上 | `入力#hitSurface` | `skin`、`cloth`、`hair`、`face`、`unknown`など |
| `住人_おろす` | `入力#移動距離` | 持ち上げた位置からの距離、logical px |
| `住人_歩き終わり` | `入力#理由` | `到着`または`中断` |
| `窓_出現` / `窓_注目` / `窓_消滅` | `入力#アプリ` | 正規化されたアプリ名 |
| 同上 | `入力#窓ID` | `枠に座る`へ渡す不透明ID |
| `フォルダ_開いた` | `入力#フォルダ` | `downloads`、`desktop`、`documents`、`pictures`、`trash`、`other` |
| 同上 | `入力#最近のダウンロード` | 直近7日の最後のファイル名、なければ空 |
| `ダウンロード_完了` | `入力#ファイル名` | フルパスではないファイル名 |
| 同上 | `入力#ファイル種類` | `image`、`video`、`audio`、`document`、`archive`、`app`、`other` |

デスクトップ観測はユーザーが明示的にONにした場合だけ届きます。観測がなくても成立するsceneを別に用意するか、届かないときは自然に沈黙する設計にします。

## 9. 表情、動作、場所を変える

### 表情と動作

```daihon
＜表情 笑顔＞
＜動作 wave＞
```

表情名はSurface・モデル側が解釈できる語彙、動作名はWorld Packの`renderer.motions`へ登録したIDを使います。未対応の表情や動作が視覚的に反映されなくても、Daihon構文自体は正しい場合があります。

### 歩く

```daihon
＜歩く 右端＞
＜歩く 左端 速さ=180 動作=walk＞
```

- 行き先: `右端`または`左端`
- `速さ`: 1秒あたりのlogical px。省略時240
- `動作`: motion ID。省略時`walk`

現在のDesktop Stageでは同一モニタ内の水平移動です。歩行完了時は`住人_歩き終わり`が届きます。

### 意味上の場所と在席

```daihon
＜場所 「pictures」＞
＜退場＞
＜退場 行き先=「downloads」＞
＜登場＞
＜登場 場所=「desktop」＞
```

`場所`はOSの実パスや画面座標ではなく、物語上のlocation IDです。

- `場所`: 現在地だけ変え、姿は消さない。
- `退場`: 現在地を保ったまま`away`にする。
- `退場 行き先=...`: 場所変更と退場を同時に行う。
- `登場`: 現在地を保ったまま`present`にする。
- `登場 場所=...`: 場所変更と登場を同時に行う。

退場中の住人は、現在のDesktop Surfaceで身体、吹き出し、選択肢、音声が提示されません。状態は保存されるため、再起動だけで勝手に戻りません。必ず戻るsceneも設計してください。

```daihon
## フォルダ_開いた

### Downloadsで遭遇
合図: ＠フォルダ_開いた
条件:（入力#フォルダ = 「downloads」 かつ 入力#場所 = 入力#フォルダ かつ 入力#在席 = 「away」）
話者: ア
＜登場＞
「あっ。見つかってしまいました。」
```

### ウィンドウの枠に座る

```daihon
## 窓_出現

### 新しい窓へ座る
合図: ＠窓_出現
話者: ア
＜枠に座る (入力#窓ID)＞
「ここは見晴らしがいいですね。」
```

降りるときは`＜枠から降りる＞`です。対象ウィンドウが閉じた場合も自動で降ります。`入力#窓ID`は不透明なIDなので、文字列を解析しないでください。

## 10. ユーザーに選んでもらう

`選択`はAIを使わず、吹き出しへ2〜6個のボタンを出します。

```daihon
## お誘い

### 散歩へ誘う
話者: ア
「少し散歩しませんか？」
_返事 = ＜選択 「行く」 「あとで」 秒数=60＞

※（_返事 = 「行く」）なら:
「では、出発します。」
＜歩く 右端＞
※あるいは（_返事 = 「あとで」）:
「分かりました。また誘います。」
※それ以外:
「今は忙しそうですね。」
おわり
```

- 選択肢は2〜6個。
- `秒数`は5〜600秒。省略時30秒。
- タイムアウト時は`不明`。
- `※それ以外`など、`不明`を処理する枝が必須。
- 1回のevent dispatchにつき最大4回。

待っている間の会話イベントはキューされ、選択終了後に処理されます。

## 11. AIを台本の中で使う

AI機能はExtension capabilityです。providerが未導入、無効、timeout、失敗のどの場合でも台本が破綻しないよう、`不明`やフォールバックを必ず設計します。

### `解釈`: 選択肢へ分類する

```daihon
_意図 = ＜解釈 (入力#ユーザー発言) 「発言の意図は何ですか」 「挨拶/質問/終了」＞

※（_意図 = 「挨拶」）なら:
「こんにちは。」
※あるいは（_意図 = 「質問」）:
「質問ですね。」
※あるいは（_意図 = 「終了」）:
「またあとで。」
※それ以外:
「うまく読み取れませんでした。」
おわり
```

`解釈`は文章を生成せず、指定した選択肢または`不明`を返します。引数は次の順です。

1. 解釈対象
2. AIへ渡す質問
3. `/`、`／`、`|`、`、`、`,`のいずれかで区切った選択肢

### `抽出`: 短い値を取り出す

```daihon
_呼び名 = ＜抽出 (入力#ユーザー発言) 「ユーザーが呼んでほしい名前だけ」＞

※（_呼び名 = 「不明」）なら:
「うまく聞き取れませんでした。」
※それ以外:
全体#呼び名 = _呼び名
「＜全体#呼び名＞さん、ですね。覚えました。」
おわり
```

結果は100文字までの文字列または`不明`です。保存スコープへ代入する前に確認してください。AIが返した値を無条件でファイルパス、URL、コマンドなどへ使う設計にはしないでください。

### `生成`: scene内の一文を作る

```daihon
＜生成 「朝の気分を短く一言で話す」 「今日も静かな朝ですね。」＞
```

第1引数は指示、第2引数は任意のフォールバックです。入力を含める場合は文字列を結合します。

```daihon
＜生成 (「前面アプリは 」 + 入力#アプリ + 「。窓として30文字以内で描写する」) (入力#アプリ + 「の窓が前に来ました。」)＞
```

providerがない、失敗した、`speak: false`だった、空文字だった場合はフォールバックを表示します。フォールバックもなければ、その行だけを飛ばしてsceneを続けます。

`解釈`と`抽出`は1 dispatchにつき合計4回までです。`生成`は別枠で1 dispatchにつき4回までです。AIを細かなループの代わりに何度も呼ぶのではなく、1回の判断・1回の短文へまとめます。

## 12. sceneをつなぐ

同じイベント内の別sceneへ移動できます。

```daihon
## チュートリアル

### はじめ
話者: ア
「説明を始めます。」
→つづき

### つづき
話者: ア
「これが続きです。」
→おわり
```

| 書き方 | 意味 |
| --- | --- |
| `→シーン名` | 同じイベント内のsceneへ移る |
| `→シーンおわり` | 現在sceneだけを終了 |
| `→おわり` | イベント全体を終了 |

イベントをまたぐジャンプはできません。循環ジャンプが多すぎると、無限ループ防止の実行エラーになります。

1行条件でジャンプすることもできます。

```daihon
※（全体#初対面済み = はい）→いつもの挨拶
```

複雑になったら、通常の`※（...）なら:`ブロックの方が読みやすくなります。

## 13. コメントを書く

`$$`または`＄＄`から行末までがコメントです。

```daihon
$$ 初回だけ見せる自己紹介
### 初対面
頻度: 一度きり
話者: ア
「はじめまして。」 $$ この部分もコメント
```

作者の意図、必要な入力、素材ID、fallbackの理由を書いておくと、数か月後の修正が楽になります。

## 14. 実践: 小さな生活サイクルを作る

次は、起動、触れ合い、外出、再会までを含む小さな例です。`pack.json`のallowlistには`app.startup`、`avatar.gesture.poke`、`presence.life_tick`、`desktop.folder.opened`を入れてください。

```daihon
## アプリ_起動

初期値:
全体#初対面済み = いいえ
全体#外出回数 = 0

### 初対面
合図: ＠アプリ_起動
条件:（全体#初対面済み = いいえ）
話者: ア
全体#初対面済み = はい
＜表情 笑顔＞
「はじめまして。今日から、ここで暮らします。」

### 再会
合図: ＠アプリ_起動
話者: ア
「おかえりなさい。」

## 住人_つつく

### 頭
合図: ＠住人_つつく
条件:（入力#hitZoneId = 「head」）
話者: ア
「髪を整えていたところです。」

### その他
合図: ＠住人_つつく
話者: ア
「呼びましたか？」

## 生活_定期

### Downloadsへ出かける
合図: ＠生活_定期
条件:（入力#在席 = 「present」 かつ 13:00~16:00）
頻度: 1日に1回
話者: ア
全体#外出回数 = 全体#外出回数 + 1
「Downloadsを見てきます。」
＜退場 行き先=「downloads」＞

### 自分で帰る
合図: ＠生活_定期
条件:（入力#在席 = 「away」 かつ 入力#場所 = 「downloads」）
話者: ア
＜登場 場所=「desktop」＞
「戻りました。今日で＜全体#外出回数＞回目の探検です。」

## フォルダ_開いた

### Downloadsで見つかる
合図: ＠フォルダ_開いた
条件:（入力#フォルダ = 「downloads」 かつ 入力#場所 = 入力#フォルダ かつ 入力#在席 = 「away」）
話者: ア
＜登場＞
「あっ。こんなところで会うとは思いませんでした。」
```

この例で大切なのは、退場させたら「生活tickで帰る」と「ユーザーが同じ場所を開いて出会う」の2つの帰還経路があることです。特別演出だけでなく、住人が戻れなくなる状態も考えて台本を書きます。

## 15. テストするときの順序

1. `pack.json`へscriptとsignalを追加する。
2. Daihonへ条件なしの最小sceneを書く。
3. CLIでsignalを起こし、セリフが出るか確認する。
4. `条件:`を1つ追加する。
5. `入力#...`をセリフへ一時表示し、実値を確認する。
6. 変数更新を追加し、snapshotや再起動後の挙動を確認する。
7. デスクトップ版でVRM・吹き出し・動作を確認する。
8. AI・TTSは最後に有効化し、providerなしのfallbackも確認する。

条件が合わないとき、いきなり複雑な式を直すより、一時的に次のsceneを作ると入力を確認しやすくなります。

```daihon
### デバッグ用
合図: ＠フォルダ_開いた
話者: ア
「場所=＜入力#場所＞、在席=＜入力#在席＞、フォルダ=＜入力#フォルダ＞」
```

配布前には削除するかコメントアウトしてください。詳しい確認手順は [テスト・配布・トラブルシューティング](04-testing-and-distribution.md) にあります。

## 16. 初心者がつまずきやすい点

| 問題 | 原因 | 修正 |
| --- | --- | --- |
| セリフが出ない | signalがallowlistにない | `pack.json`の`signals.allow`へcanonical IDを追加 |
| sceneが選ばれない | `合図:`の`＠`忘れ、条件が偽、頻度制限中 | 診断とイベント入力を確認 |
| 別sceneばかり出る | より具体的な条件のsceneが優先 | 条件数と受け皿sceneを見直す |
| `話者`がunknown | actor IDでもaliasでもない | `speakerAliases`または台本を修正 |
| 変数の型エラー | 数値へ文字列などを代入 | 初期値と全代入箇所の型をそろえる |
| `入力#...`へ代入できない | 入力は読み取り専用 | `_一時`や`全体#...`へコピーする |
| 選択・解釈で構文エラー | `不明`を受ける枝がない | `※それ以外`を追加 |
| AIなしで無反応 | fallbackがない、余白委任だけに依存 | 固定sceneや`生成`の第2引数を追加 |
| 退場後に姿が戻らない | 登場sceneがない | tick、フォルダ遭遇、起動などに帰還経路を追加 |
| `優先度:`が効かない | 廃止済み | より具体的な条件を書く |

構文の全項目、演算子、診断コードは [Daihon言語リファレンス](../../08-daihon-language-reference.md) を参照してください。
